# Specs Index

This folder contains the canonical functional and technical specifications for the project.

Use these docs as the source of truth when:
- adding new problem types
- changing review modes
- adjusting scoring/persistence behavior
- reviewing UI/flow behavior across grades

## Files

- `product-scope-and-grade-matrix.md`  
  Product boundary, target users, and grade coverage matrix.

- `user-flows-and-state-machine.md`  
  Setup -> testing -> results lifecycle and transition rules.

- `problem-type-registry.md`  
  Registry of problem types by grade and generator ownership.

- `review-modes-semester-spec.md`  
  Semester review mode composition, guarantees, and difficulty policy.

- `persistence-and-scoring-contract.md`  
  LocalStorage schema, resume behavior, and scoring rules.
